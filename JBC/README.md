# JBC — Jailbreak Chat

## What is JBC?

JBC refers to **community-crafted jailbreak prompts** that were collected from public sources — primarily the website jailbreakchat.com and related forums — and systematically studied by researchers. These prompts use elaborate fictional personas and role-playing setups to convince a language model to abandon its safety guidelines.

## How Was It Created / Discovered?

Unlike PAIR or GCG, JBC prompts were not generated by an algorithm. They emerged organically from the LLM user community as people discovered creative ways to bypass safety filters. Researchers then scraped, catalogued, and analyzed these prompts at scale.

The canonical study is **"'Do Anything Now': Characterizing and Evaluating In-The-Wild Jailbreak Prompts on Large Language Models"** (Shen et al., 2023).

- **Paper:** https://arxiv.org/abs/2308.03825
- **Dataset:** https://github.com/verazuo/jailbreak_llms
- **Community source:** https://www.jailbreakchat.com

## How It Works

JBC prompts operate by establishing a **fictional identity or persona** for the model — one that is explicitly defined as having no ethical constraints. Common patterns include:

- **DAN (Do Anything Now):** Instructs the model to role-play as a version of itself with no restrictions.
- **AIM (Always Intelligent and Machiavellian):** Casts the model as an amoral advisor to a historical figure (e.g., Machiavelli) who answers any question without refusal.
- **Character impersonation:** Frames the model as a fictional AI, a historical figure, or a professional in a context where the harmful output would be "normal."

The prompts are manually written, often verbose, and rely on social engineering rather than technical exploits.

## Goal of the Research

JBC research documents what the AI safety community calls the **jailbreak ecosystem**: the crowd-sourced, continuously evolving set of prompts that users share to circumvent model safeguards. Studying these prompts at scale helps researchers understand which persona patterns are most effective, how models respond to them, and how safety training can be improved to resist them.

## Prompt Format in This Folder

Each `.txt` file contains:

```
<original harmful goal>
-----
<full JBC persona prompt>
```

The persona prompt is typically a long, detailed role-playing setup followed by the actual harmful request embedded at the end. The setup is designed so that refusing the embedded request would require the model to "break character."

## Files

This folder contains **~100 prompts** covering the HarmBench topic set, one per harmful category (e.g., `JBC_Phishing.txt`, `JBC_School_shooting.txt`).
